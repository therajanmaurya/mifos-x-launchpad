## Writing Guidelines

### SPEC.md Structure
```markdown
# Step N: {{Step Name}}

## Overview
Brief description of the step's purpose

## User Stories
- As a user, I want to...

## Requirements
### Functional
- FR-1: ...

### Non-Functional
- NFR-1: ...

## Acceptance Criteria
- [ ] AC-1: ...

## Edge Cases
- EC-1: ...

## Error Handling
- ERR-1: ...
```

### Quality Checklist

- [ ] Clear, concise language
- [ ] No ambiguous requirements
- [ ] All edge cases documented
- [ ] Error messages defined
- [ ] Acceptance criteria testable
- [ ] Dependencies noted
